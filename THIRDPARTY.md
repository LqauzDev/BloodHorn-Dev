# Third-Party Software Acknowledgments

This file acknowledges the third-party software, frameworks, and libraries used in the BloodHorn bootloader project. We are grateful to the authors and maintainers of these projects for their contributions to the open-source community.

## EDK2 (EFI Development Kit II)

**Description**: EDK2 is a modern, feature-rich firmware development environment based on the UEFI Specification. BloodHorn uses a modified version of EDK2 to provide UEFI services and hardware abstraction.

**Author/Maintainer**: Tianocore Project
**License**: BSD-2-Clause-Patent
**Repository**: https://github.com/tianocore/edk2
**Website**: https://www.tianocore.org/

**Usage in BloodHorn**:
- Core UEFI framework and services
- Hardware abstraction layer
- Build system and development tools
- Platform initialization and boot services

## cLib

**Description**: A lightweight, portable C standard library implementation designed for embedded systems, freestanding environments, and situations where a minimal footprint is required. Provides complete stdlib, stdio, string, math, and other standard C library functionality.

**Author**: AtomicElectron
**License**: Public Domain
**Repository**: https://codeberg/BloodyHell-Industries-INC/cLib

**Usage in BloodHorn**:
- Complete C standard library implementation
- Memory management (malloc, free, realloc, calloc)
- String manipulation and character handling
- Standard I/O operations (printf, fprintf, etc.)
- Mathematical functions
- Time and date functions
- Error handling and assertions
- Compatibility layer for EDK2 integration

## cc-runtime

**Description**: Compiler runtime compatibility library providing essential runtime functions and compiler-specific implementations for cross-platform compatibility.

**Author**: Mintsuki
**License**: Apache
**Repository**: https://codeberg.org/OSDev/cc-runtime

**Usage in BloodHorn**:
- Compiler compatibility layer
- Runtime function implementations
- Cross-platform build support
- Integration with different toolchains
